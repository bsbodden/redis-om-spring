== Introduction to Object Mapping

Redis OM Spring provides a robust object mapping framework that allows you to seamlessly map Java objects to Redis data structures. This page explains the core principles of object mapping in Redis OM Spring.

== Core Object Mapping Concepts

=== Entity Classes

Entity classes are Plain Old Java Objects (POJOs) that are annotated to indicate they should be persisted to Redis. Depending on the storage format, you'll use different annotations:

* `@RedisHash` - For mapping to Redis Hashes (standard Spring Data Redis)
* `@Document` - For mapping to Redis JSON (Redis OM Spring)

=== Entity IDs

Every entity requires a unique identifier, which can be:

* Explicitly assigned by your application
* Automatically generated by Redis OM Spring using the built-in ULID generator

[source,java]
----
@Document
public class Product {
    @Id
    private String id;  // Will be automatically generated if not provided
    
    // Other fields
}
----

=== Type Conversion

Redis OM Spring handles automatic conversion between Java types and Redis storage formats:

* Java primitive types (String, Integer, Long, etc.) → Redis string values
* Java collection types (List, Set, Map) → Redis complex structures
* Java Date/Time types → Redis string representation
* Custom Java objects → Nested structures (especially for JSON)

== Field Mapping

=== Basic Field Mapping

By default, all fields of a class are mapped to Redis:

[source,java]
----
@Document
public class Customer {
    @Id
    private String id;
    private String name;         // Mapped to name field
    private Integer age;         // Mapped to age field
    private boolean active;      // Mapped to active field
    // Getters and setters
}
----

=== Ignoring Fields

Fields can be excluded from persistence using the standard Jackson annotations:

[source,java]
----
@Document
public class User {
    @Id
    private String id;
    private String username;
    
    @JsonIgnore
    private String temporaryAuthToken;  // Not stored in Redis
}
----

=== Custom Field Names

You can customize how fields are named in Redis:

[source,java]
----
@Document
public class Product {
    @Id
    private String id;
    
    @JsonProperty("product_name")
    private String name;
    
    @JsonProperty("unit_price")
    private double price;
}
----

== Advanced Mapping Features

=== Embedded Objects

Redis OM Spring supports mapping of embedded objects:

[source,java]
----
@Document
public class Order {
    @Id
    private String id;
    private Customer customer;      // Embedded object
    private List<OrderItem> items;  // Collection of embedded objects
}
----

=== Inheritance

Redis OM Spring supports entity inheritance:

[source,java]
----
@Document
public abstract class Vehicle {
    @Id
    private String id;
    private String manufacturer;
}

@Document
public class Car extends Vehicle {
    private int numberOfDoors;
    private String engineType;
}
----
